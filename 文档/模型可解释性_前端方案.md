# 模型可解释性展示 · 前端实现方案

> 目标：将模型可解释性展示作为**花卉识别页 (`RecognitionView.vue`) 的下翻第二屏**，用户在识别页向下滚动即可看到 ResNet50 的 4 类可视化分析。不新增路由，不新增页面文件。
>
> 技术栈：Vue 3 + TypeScript + Composition API + Element Plus + ECharts（项目已集成）。
> 滚动方案：CSS Scroll Snap（原生、零依赖、平滑过渡）。

---

## 一、整体交互设计

```
┌────────────────────────────────────┐
│         顶部导航栏 (3.75rem)         │  ← 不变
├────────────────────────────────────┤
│                                    │
│     第 ① 屏：花卉识别 (现有)          │  ← scroll-snap-align: start
│     上传图片 → 点击识别 → 结果展示     │
│                                    │
│          ⌄ 向下滚动 ⌄               │  ← 滚动指示器
├────────────────────────────────────┤  ← snap 边界
│                                    │
│     第 ② 屏：模型可解释性 (新增)      │  ← scroll-snap-align: start
│     [结构] [训练曲线] [Grad-CAM]    │
│     [特征降维]  ← el-tabs           │
│                                    │
│     <各 Tab 内容>                  │
└────────────────────────────────────┘
```

- 第一屏 = 现有花卉识别功能（保持不变）
- 第二屏 = 新增模型可解释性分析（4 个 Tab）
- 两屏之间通过 `scroll-snap` 实现整页吸附，滚动丝滑无割裂感
- 第二屏内部 Tab 切换时，该屏高度自适应内容（不强制整屏，内容多时可继续滚动）

## 三、RecognitionView.vue 改造

### 3.1 改造思路

将现有 `RecognitionView.vue` 的根容器从单屏布局改为**双屏滚动容器**：

- 外层 `.scroll-container`：`height: calc(100vh - 3.75rem)` + `overflow-y: scroll` + `scroll-snap-type: y mandatory`
- 第一屏 `.snap-page.recognition-screen`：`scroll-snap-align: start`，包裹现有全部识别逻辑（原 `<template>` 内容原样移入）
- 第二屏 `.snap-page.explain-screen`：`scroll-snap-align: start`，嵌入 `<ModelExplainPanel />`
- 底部滚动指示器：第一屏底部显示 "↓ 模型可解释性分析"，点击平滑滚动到第二屏

### 3.3 改造注意事项

| 事项 | 说明 |
|------|------|
| 原有逻辑保留 | `onFileChange` / `onDrop` / `loadFile` / `reset` / `identify` 等函数和响应式变量**全部保留不变**，仅外层 DOM 结构包裹到 `.snap-page` 中 |
| 背景图迁移 | 原根容器 `.recognition-page` 上的背景图样式迁移到 `.recognition-screen` |
| 高度计算 | `.scroll-container` 高度 = `calc(100vh - 3.75rem)`（减去导航栏 3.75rem），与原 `.recognition-page` 的 `min-height` 一致 |
| 第二屏高度 | `.explain-screen` 用 `min-height: 100%` 而非固定 `100vh`，当 Tab 内容超出视口时可自然向下延伸（此时滚动 snap 只在第一屏→第二屏边界生效） |
| `scroll-snap-stop: always` | 确保快速滚动时不会跳过第一屏直接到第二屏 |

## 五、API 封装 `src/api/model.ts`

```ts
import { apiRequest, get } from '@/utils/request'

const MODEL_API_BASE = 'http://localhost:5000'

// ============ 类型定义 ============

/** 模型结构层节点 */
export interface ModelLayer {
  name: string
  type: string
  out_shape: number[]
  params: number
  trainable: boolean
  detail?: string
  children?: ModelLayer[]
}

export interface ModelStructure {
  model_type: string
  total_params: number
  trainable_params: number
  input_shape: number[]
  output_classes: number
  layers: ModelLayer[]
}

/** 训练曲线 */
export interface TrainingHistory {
  model_type: string
  total_epochs: number
  epochs: number[]
  train_loss: number[]
  val_loss: number[]
  train_acc: number[]
  val_acc: number[]
  lr?: number[]
}

/** Grad-CAM 结果 */
export interface GradCAMResult {
  original: string   // data:image/png;base64,...
  heatmap: string
  overlay: string
  predicted_class: number
  predicted_name: string
  target_class: number
  top5: { class_id: number; class_name: string; prob: number }[]
  target_layer: string
  alpha: number
}

/** t-SNE 散点 */
export interface FeaturePoint {
  x: number
  y: number
  class_id: number
  class_name: string
  sample_index: number
}
export interface FeatureClass {
  class_id: number
  class_name: string
  count: number
  centroid_x: number
  centroid_y: number
}
export interface FeatureData {
  method: string
  n_samples: number
  points: FeaturePoint[]
  classes: FeatureClass[]
}

// ============ API 函数 ============

export function getModelStructure() {
  return get<ModelStructure>(`${MODEL_API_BASE}/api/model/structure`)
}

export function getTrainingHistory() {
  return get<TrainingHistory>(`${MODEL_API_BASE}/api/model/training`)
}

export function getGradCAM(params: {
  image?: File
  sampleIndex?: number
  targetLayer?: string
  targetClass?: number
  alpha?: number
}) {
  const formData = new FormData()
  if (params.image) formData.append('image', params.image)
  if (params.sampleIndex !== undefined) formData.append('sample_index', String(params.sampleIndex))
  if (params.targetLayer) formData.append('target_layer', params.targetLayer)
  if (params.targetClass !== undefined) formData.append('target_class', String(params.targetClass))
  if (params.alpha !== undefined) formData.append('alpha', String(params.alpha))
  return apiRequest<GradCAMResult>(`${MODEL_API_BASE}/api/model/gradcam`, formData, { timeout: 30000 })
}

export function getFeatures(params: { nSamples?: number; perplexity?: number; method?: 'tsne' | 'pca' }) {
  const q = new URLSearchParams()
  if (params.nSamples) q.set('n_samples', String(params.nSamples))
  if (params.perplexity) q.set('perplexity', String(params.perplexity))
  if (params.method) q.set('method', params.method)
  return get<FeatureData>(`${MODEL_API_BASE}/api/model/features?${q.toString()}`)
}
```

> 注意：参考现有 `api/flower.ts` 的写法，`apiRequest` 和 `get` 从 `@/utils/request` 导入。需确认 `get<T>` 是否自动解包 `.data` 字段——若后端返回 `{code, message, data}`，`get` 可能返回整个外层对象。**联调时确认返回层级，必要时调整**。

```ts
      lineStyle: { type: 'dashed' }, itemStyle: { color: '#2ECC71' } },
    { name: 'Val Acc',    type: 'line', data: smooth(history.val_acc, window), yAxisIndex: 1,
      itemStyle: { color: '#2ECC71' } },
  ]
}

// 平滑函数（前端实现，避免重复请求）
function smooth(arr: number[], window: number): number[] {
  if (window <= 1) return arr
  return arr.map((_, i) => {
    const s = Math.max(0, i - Math.floor(window/2))
    const e = Math.min(arr.length, s + window)
    return arr.slice(s, e).reduce((a,b)=>a+b,0) / (e-s)
  })
}
```

## 十、联调注意事项

### 10.1 后端响应层级

后端统一返回 `{ code, message, data }`。需确认 `@/utils/request` 的 `get<T>` 返回的是 `T`（已解包 `data`）还是外层对象。参考 `api/flower.ts` 中 `identifyFlower` 直接用 `apiRequest<FlowerIdentifyResult>` 拿到解包后的结果——**4 个 API 同理处理**：

- `getModelStructure` / `getTrainingHistory` / `getFeatures` 用 `get<T>`（需确认是否自动解包）
- `getGradCAM` 用 `apiRequest<T>`（与 `identifyFlower` 一致）

若 `get` 不解包，改为：

```ts
const res = await get<{ code:number; data: ModelStructure }>(url)
structure.value = res.data
```

### 10.2 CORS

后端已全局 `CORS(app)`，前端跨域调用 `localhost:5000` 无需额外配置。

### 10.3 加载状态

| 接口 | 预期耗时 | 处理 |
|------|---------|------|
| structure | < 1s | 页面骨架 |
| training | < 100ms | 骨架 |
| gradcam | 1~3s | loading 遮罩 + 按钮禁用 |
| features (tsne) | 5~15s | **全屏 loading** + 文案"正在计算 t-SNE，请稍候..." |

### 10.4 错误处理

参考 `RecognitionView.vue` 的 `try/catch + alert` 模式。t-SNE 超时时提示用户减少采样数或改用 PCA。

### 10.5 滚动与 ECharts 冲突

第二屏内 ECharts 图表的 `dataZoom`（滚轮缩放）可能与外层 `scroll-container` 的滚动产生冲突。解决方案：

- **方案 A（推荐）**：ECharts 图表区域设置固定高度 + `overflow: auto`，将滚轮事件限制在图表内部
- **方案 B**：ECharts `dataZoom` 只用 `type: 'slider'`（拖拽滑块），不用 `type: 'inside'`（滚轮），避免与页面滚动冲突
- **方案 C**：第二屏不用 `scroll-snap`，改为普通滚动，仅第一屏→第二屏的切换用 JS 控制平滑滚动
