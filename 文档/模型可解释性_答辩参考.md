### 2.3 答辩要点

**Q: 为什么 ResNet50 能处理 102 类花卉分类？**

> ResNet50 是在 ImageNet（1000 类，1400 万张图片）上预训练的模型。我们通过迁移学习，冻结前面 44 层（提取通用视觉特征），只微调最后的全连接层（2048→512→256→102），使其适应 102 类花卉任务。总参数量 2470 万，可训练参数仅 120 万。

**Q: 什么是残差块（Bottleneck）？为什么叫"残差"？**

> 传统网络只学 F(x)，ResNet 学 F(x) + x（恒等映射跳跃连接）。这让梯度能直接回传到浅层，解决了深层网络的梯度消失问题，使得训练 50 层以上的网络成为可能。Bottleneck 结构是 1×1 降维 → 3×3 卷积 → 1×1 升维，比普通卷积更高效。

**Q: 为什么 layer3 有 6 个 Bottleneck 而 layer1 只有 3 个？**

> 越深层需要越多的残差块来学习复杂的特征组合。浅层（layer1）学习简单的边缘和纹理，不需要太多块；深层（layer3）需要组合这些低级特征形成"花瓣形状""花蕊结构"等高级语义，所以需要更多块。

| **Val Accuracy 最终值** | 模型性能指标 | 最终 96.08%，非常优秀 |

### 3.3 答辩要点

**Q: 为什么分两个阶段训练？**

> 这是迁移学习的标准做法：
> - **第一阶段（冻结训练）**：冻结 ResNet50 前 44 层，只训练最后的全连接层。学习率较大（0.001），快速让分类头适应花卉数据。这一步很快（约 13 分钟），因为只需更新 120 万参数。
> - **第二阶段（微调）**：解冻 layer3 和 layer4，用小学习率（0.0001）精细调整。让模型在已学特征的基础上，针对花卉数据做微调。
>
> 如果直接全量训练，容易在小数据集上过拟合，且训练时间很长。

**Q: 什么是早停（Early Stopping）？为什么需要它？**

> 当验证集 Loss 连续 N 轮（patience=10/5）没有显著下降时，自动停止训练。防止模型在训练集上继续优化导致过拟合——即训练 Loss 继续降但验证 Loss 开始上升。

**Q: 训练曲线中 Loss 先降后平，但 Accuracy 持续上升，为什么不同步？**

> Loss 是连续值（交叉熵），Accuracy 是离散值（对/错）。当模型对大部分样本已经很确信时，Loss 变化很小，但那些之前判错的边界样本仍在被逐步纠正，所以 Accuracy 还能继续提升。

## 四、模块 3：Grad-CAM 热力图

### 4.1 实现原理

| 项目 | 说明 |
|------|------|
| 算法 | Grad-CAM（Gradient-weighted Class Activation Mapping） |
| 核心思想 | 用目标类别的梯度加权卷积层的特征图，生成类别的注意力热力图 |
| 公式 | `L = ReLU(Σ_k α_k · A_k)`，其中 `α_k = mean(∂y/∂A_k)` |
| 关键代码 | `api_model.py` 中 `GradCAM` 类，hook 注册到指定卷积层 |

**算法流程**：

```
输入图片 → ResNet50 forward → 得到预测类别 y
                ↓
对目标类别 y 做 backward → 得到梯度 ∂y/∂A_k
                ↓
梯度全局平均池化 → 权重 α_k
                ↓
权重 × 特征图 → 加权求和 → ReLU → 热力图
                ↓
上采样到 224×224 → 与原图叠加 → 可视化
```

### 4.2 图表解读

页面展示三张图：

| 图片 | 含义 | 解读方式 |
|------|------|---------|
| **原图** | 输入的花卉图片 | 识别前的原始图片 |
| **热力图** | 模型的注意力分布 | 🔴红色=高关注，🔵蓝色=低关注 |
| **叠加图** | 热力图叠加到原图 | 直观看模型"看哪里"做判断 |

**解读示例**：

```
好的情况（模型看对了）：           坏的情况（模型看错了）：
┌──────────────┐                ┌──────────────┐
│   🌸 花朵     │                │   🌸 花朵     │
│  ┌──────┐    │                │              │
│  │🔴 红色│    │                │        🔴    │  ← 关注背景而非花朵
│  │ 花瓣  │    │                │       背景    │
│  └──────┘    │                │              │
│  蓝色背景    │                │  蓝色花朵    │
└──────────────┘                └──────────────┘
  模型正确关注花朵                 模型关注了错误区域
```

### 4.3 不同目标层的含义

| 目标层 | 特征级别 | 热力图特点 | 适用场景 |
|--------|---------|-----------|---------|
| **Layer 1** | 低级特征 | 关注边缘、纹理、颜色块 | 查看模型是否在看"像素级"特征 |
| **Layer 2** | 中级特征 | 关注局部图案、花瓣边缘 | 查看模型是否在看"部件级"特征 |
| **Layer 3** | 高级特征 | 关注花朵整体形状 | 查看模型是否在看"物体级"特征 |
| **Layer 4** | 最高级特征 | 关注花朵核心区域 | 查看模型的最终决策依据 |

### 4.4 答辩要点

**Q: Grad-CAM 的数学原理是什么？**

> 1. 对目标类别的得分 y^c 关于特征图 A^k 求梯度：∂y^c/∂A^k
> 2. 对梯度做全局平均池化，得到每个通道的重要性权重：α_k^c = (1/Z) Σ_i Σ_j (∂y^c/∂A_{ij}^k)
> 3. 用权重对特征图加权求和，再过 ReLU：L^c = ReLU(Σ_k α_k^c · A^k)
> 4. 上采样到原图尺寸，叠加显示

**Q: 为什么选择 layer4 而不是其他层？**

> Layer4 是最后一个卷积层，输出 2048 个 7×7 的特征图，包含最高级的语义信息。它最接近分类决策，所以其热力图最能反映模型"基于什么做最终判断"。浅层热力图更关注纹理，深层更关注语义。

**Q: 热力图能发现什么问题？**

> - **关注区域正确**：模型看花瓣、花蕊等判别性区域 → 说明学到了正确特征
> - **关注背景**：模型看花盆、背景草地 → 说明存在"捷径学习"，可能数据有偏
> - **关注分散**：热力图没有明显焦点 → 说明模型对这个类别不够确信

## 五、模块 4：t-SNE 特征降维

### 5.1 实现原理

| 项目 | 说明 |
|------|------|
| 算法 | t-SNE（t-distributed Stochastic Neighbor Embedding） |
| 核心思想 | 保持高维空间中的"邻近关系"，映射到 2D 平面 |
| 流程 | 提取 layer4 特征（2048 维）→ Global Avg Pool → t-SNE 降维到 2D |
| 关键代码 | hook 提取特征 → sklearn TSNE → 前端 ECharts 散点图 |

**算法流程**：

```
测试集图片 → ResNet50 forward → hook 提取 layer4 输出
                ↓
Global Average Pooling → 2048 维特征向量
                ↓
t-SNE 降维（2048D → 2D）
                ↓
按类别着色绘制散点图
```

### 5.2 图表解读

```
        t-SNE dim2
         ↑
    10 ──┤     ●●●           ■■■
         │    ●rose●        ■daisy■
     5 ──┤     ●●●           ■■■
         │
     0 ──┤          ▲▲▲
         │         ▲tulip▲
    -5 ──┤          ▲▲▲     ◆◆◆
         │                ◆lily◆
   -10 ──┤                 ◆◆◆
         └──┼──┼──┼──┼──┼──┼──→ t-SNE dim1
          -10  -5   0   5   10
```

**解读要点**：

| 现象 | 含义 | 好/坏 |
|------|------|-------|
| **同类聚集** | 同一花朵的特征向量相似 | ✅ 好，说明模型学到了判别性特征 |
| **异类分离** | 不同花朵的特征分开 | ✅ 好，说明模型能区分不同类别 |
| **类别重叠** | 两类花的点混在一起 | ⚠️ 需关注，可能是视觉相似的花 |
| **离群点** | 某个点远离其类别群 | ⚠️ 可能是误分类或特殊样本 |
| **所有点混在一起** | 无法区分 | ❌ 差，说明特征提取失败 |

### 5.3 t-SNE vs PCA

| 对比项 | t-SNE | PCA |
|--------|-------|-----|
| 降维方式 | 非线性，保持局部邻近关系 | 线性，保持全局方差 |
| 速度 | 慢（5-15 秒） | 快（<1 秒） |
| 可视化效果 | 聚类更明显 | 可能看不出聚类 |
| 可复现性 | 每次结果略有不同（随机初始化） | 结果固定 |
| 适用场景 | 探索性分析、展示聚类 | 快速预览、特征选择 |

### 5.4 答辩要点

**Q: t-SNE 的数学原理是什么？**

> 1. 在高维空间中，用高斯分布计算每个点对的相似度（条件概率）
> 2. 在低维空间中，用 t 分布计算对应点对的相似度
> 3. 最小化两个分布的 KL 散度，使低维空间保持高维的邻近关系
> 4. t 分布的重尾特性解决了"拥挤问题"，使得不同类别能被分开

**Q: 为什么选择 layer4 的特征而不是其他层？**

> Layer4 是最后一个卷积层，输出的 2048 维特征最接近分类决策，包含了最高级的语义信息（花朵形状、颜色组合等）。浅层特征（如 layer1 的 64 维）主要编码边缘和纹理，无法有效区分不同花卉类别。

**Q: t-SNE 结果每次不同，这正常吗？**

> 正常。t-SNE 使用随机初始化，每次运行结果略有不同，但整体聚类结构应该一致。如果聚类结构变化很大，说明数据分布不够清晰。PCA 是确定性算法，结果固定。

**Q: 如果看到两类花聚在一起，说明什么？**

> 说明这两类花在特征空间中非常相似，模型难以区分。可能原因：
> 1. 视觉上确实相似（如不同品种的玫瑰）
> 2. 训练数据不足，模型没有学到足够的判别特征
> 3. 需要更深层的特征或更好的数据增强
